Angular + PocketBase + Gotenberg

Documentación técnica de VerificarIT

VerificarIT es una PWA para crear, consultar, editar y exportar inspecciones vehiculares. El proyecto está depurado para entrega y conserva únicamente páginas, servicios y flujos activos.

Repositorio GitHub README Schema PocketBase
FrontendAngular 21 standalone
BackendPocketBase v0.19.0
ReportesExcelJS + Gotenberg
EstadoCódigo depurado

Arquitectura

La aplicación es un frontend estático. Angular consume PocketBase y un endpoint de PDF definidos por configuración runtime; no existe backend Node dentro del repositorio.

Modelo De Datos

El schema de PocketBase está versionado en docs/pb_schema.json y corresponde a PocketBase server v0.19.0. La app activa consume principalmente users, inspections, images y secuencias.

ColecciónUsoCampos relevantes
usersAutenticaciónname, avatar, type, role, phone, status.
inspectionsInspecciones activas117 campos para conductor, propietario, vehículo, documentos, checklist, firmas, QR y verificación.
imagesFotografíasCampo file image; los IDs se guardan en inspections.images.
secuenciasConsecutivosultimo_numero, tipo_inspeccion, prefijo.
filesArchivos auxiliaresimage, type, userId.
firmasFirmasimage, numero_certificado.
inspections_busetasLegado backend98 campos históricos; el componente Angular de busetas fue retirado para la entrega.

Grupos De Campos De Inspección

  • Identificación: `id`, `numero_certificado`, `created`, `updated`.
  • Conductor y propietario: nombres, identificación, teléfono, WhatsApp, número y vigencia de licencia, propietario y documento.
  • Vehículo: placa, marca, modelo, color, clase, código, capacidad y kilometraje.
  • Documentos: SOAT, licencia de tránsito, revisión tecnomecánica y tarjeta de operación.
  • Checklist: eléctrico, motor, carrocería, cabina, seguridad, kit, parte baja, frenos, dirección y llantas.
  • Evidencias y verificación: firmas base64, imágenes asociadas, publicUrl, verificationCode y qrImage.

Realtime Y Caché

`RealtimeInspectionsService` usa `subscribe('*')` sobre `inspections`, mantiene observables para lista, eventos y errores, y cachea inspecciones en `localStorage`.

  • Clave de caché: `inspections_cache`.
  • TTL: 5 minutos.
  • Los listados excluyen firmas base64.
  • La carga completa puede ejecutarse en segundo plano.
  • La caché se invalida al crear, actualizar o eliminar.

Despliegue

Frontend En Amplify

    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*

Configurar rewrite SPA: `/<*>` hacia `/index.html` con estado 200.

PocketBase En EC2

  • Usar PocketBase server v0.19.0 para mantener compatibilidad con docs/pb_schema.json.
  • Ejecutar como servicio systemd.
  • Servir detrás de HTTPS.
  • Persistir y respaldar `pb_data`.
  • Importar o validar docs/pb_schema.json antes de liberar.
  • Versionar schema y reglas después de cada cambio operativo.
  • Restringir CORS.

Gotenberg En Dokploy

  • Imagen recomendada: `gotenberg/gotenberg:8`.
  • Dominio HTTPS.
  • Basic Auth o protección equivalente.
  • Límite de request suficiente para XLSX con imágenes.
  • Preferible usar backend intermedio para ocultar credenciales.

Seguridad Pendiente

Todo secreto incluido en Angular queda expuesto en el navegador. Gotenberg no debería autenticarse directamente desde frontend en producción.
  • Rotar credenciales expuestas históricamente en PocketBase, Gotenberg, Dokploy y proxies.
  • Rotar específicamente el login demo david@appverificar.site, la contraseña appverificar0126 y el Basic Auth de Gotenberg tremaine.bogisich-reichert / zdbdb0zoq2nqy73t.
  • Invalidar cuentas demo, contraseñas compartidas y tokens antiguos.
  • Mantener URLs en configuración runtime y secretos en infraestructura segura.
  • Versionar reglas de PocketBase.
  • Aplicar CORS estricto.
  • Agregar control server-side para permisos críticos.
  • Auditar assets demo en `public/assets`.
