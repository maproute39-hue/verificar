Angular + PocketBase + Gotenberg

Documentación técnica de VerificarIT

VerificarIT es una PWA para crear, consultar, editar y exportar inspecciones vehiculares. El proyecto está depurado para entrega y conserva únicamente páginas, servicios y flujos activos.

FrontendAngular 21 standalone
BackendPocketBase HTTPS
ReportesExcelJS + Gotenberg
EstadoCódigo depurado

Arquitectura

La aplicación es un frontend estático. Angular consume PocketBase y Gotenberg directamente desde el navegador; no existe backend Node dentro del repositorio. 

Navegador/PWA
  |
  |-- Angular 21 standalone
  |   |-- pages: login, home, nueva, heredada, inspections, detail
  |   |-- services: auth, inspection, realtime, excel, gotenberg, pwa
  |   `-- public/assets/templates/inspection.xlsx
  |
  |-- PocketBase
  |   |-- users
  |   |-- inspections
  |   |-- images
  |   `-- secuencias
  |
  `-- Gotenberg
      |-- /forms/libreoffice/convert
      `-- /forms/chromium/convert/html
Los secretos de Gotenberg y credenciales demo están actualmente en frontend. Para producción deben rotarse y moverse fuera del bundle Angular.

Modelo De Datos

El schema de PocketBase no está versionado en el repositorio. El modelo documentado se infiere desde servicios, formularios y `inspection.model.ts`.

ColecciónUsoCampos relevantes
usersAutenticación`email`, `username`, `name`, `role`, `phone`, `avatar`, `verified`.
inspectionsInspeccionesConductor, propietario, vehículo, documentos, checklist, firmas e imágenes.
imagesArchivosCampo file `image`; los IDs se referencian desde inspecciones.
secuenciasConsecutivos`prefijo`, `ultimo_numero`.

Grupos De Campos De Inspección

  • Identificación: `id`, `numero_certificado`, `created`, `updated`.
  • Conductor y propietario: nombres, identificación, teléfono, WhatsApp, licencia, propietario y documento.
  • Vehículo: placa, marca, modelo, color, clase, código, capacidad y kilometraje.
  • Documentos: SOAT, licencia de tránsito, revisión tecnomecánica y tarjeta de operación.
  • Checklist: eléctrico, motor, carrocería, cabina, seguridad, kit, parte baja, frenos, dirección y llantas.
  • Evidencias: firmas base64 e imágenes asociadas.

Realtime Y Caché

`RealtimeInspectionsService` usa `subscribe('*')` sobre `inspections`, mantiene observables para lista, eventos y errores, y cachea inspecciones en `localStorage`.

  • Clave de caché: `inspections_cache`.
  • TTL: 5 minutos.
  • Los listados excluyen firmas base64.
  • La carga completa puede ejecutarse en segundo plano.
  • La caché se invalida al crear, actualizar o eliminar.

Despliegue

Frontend En Amplify

version: 1
frontend:
  phases:
    preBuild:
      commands:
        - npm ci
    build:
      commands:
        - npm run build
  artifacts:
    baseDirectory: dist/verificar-app/browser
    files:
      - '**/*'
  cache:
    paths:
      - node_modules/**/*

Configurar rewrite SPA: `/<*>` hacia `/index.html` con estado 200.

PocketBase En EC2

  • Ejecutar como servicio systemd.
  • Servir detrás de HTTPS.
  • Persistir y respaldar `pb_data`.
  • Versionar schema y reglas.
  • Restringir CORS.

Gotenberg En Dokploy

  • Imagen recomendada: `gotenberg/gotenberg:8`.
  • Dominio HTTPS.
  • Basic Auth o protección equivalente.
  • Límite de request suficiente para XLSX con imágenes.
  • Preferible usar backend intermedio para ocultar credenciales.

Seguridad Pendiente

Todo secreto incluido en Angular queda expuesto en el navegador. Gotenberg no debería autenticarse directamente desde frontend en producción.
  • Rotar credenciales expuestas históricamente.
  • Eliminar credenciales demo del login.
  • Mover URLs y secretos a configuración segura.
  • Versionar reglas de PocketBase.
  • Aplicar CORS estricto.
  • Agregar control server-side para permisos críticos.
  • Auditar assets demo en `public/assets`.
